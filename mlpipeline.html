What is Pipeline in Machine Learning?

Published 17 April 20214 min read comments

The central activity of engineering, as distinguished from science, is the design of new devices, processes and systems.

Myron Tribus

Developing Machine Learning Models in local machines or cloud services like Kaggle, Google Collaboratory while testing and training your models in a single environment is a cool way to interact with models but unfortunately, these couldn’t be productionized. In this blog, I will be discussing what is ML pipeline, why it’s important, and what are the major challenges involved. Let’s put on our reading glasses and dive in together

What is ML pipeline?

A well-defined ML pipeline could help in automating Machine Learning workflows. Machine Learning consists of well-defined sequential steps from data collection to training to evaluating outcomes whether positive or negative. We are going to them in detail later. Machine Learning Pipeline are iterative in nature while model performs on offline data, data collection step continuously collect data to train the model at a particular time to improve efficiency and accuracy of the model.

With data being produced at a pace never seen before, we are almost creating as much data every two days as we did from the beginning of time to 2003. Tremendous value could be delivered from such humungous data tombs to companies that have led to actionable insights through today’s world. It’s not just about storing data any longer, but capturing, preserving, accessing, and transforming it to take advantage of its possibilities and the value it can deliver.

Components of ML pipeline

Mainly, a typical Machine Learning Pipeline contains following Components:

  1. Data Collection
  2. Data Validation
  3. Data Preparation
  4. Model Training
  5. Model evaluation
  6. Model Validation

Data Collection and Data Validation are one of the initial and important steps in Machine Learning Pipeline, collecting data and most importantly right data is a difficult task in itself. Mainitaning quality of data and its accessibility is one of the keys challenges building pipeline initially. If data isn’t accurate from the start, your results definitely won’t be accurate either. That’s why it’s necessary to verify and validate data before it is used.While data validation is a critical step in any data workflow, it’s often skipped over. It may seem as if data validation is a step that slows down your pace of work, however, it is essential because it will help you create the best results possible.

Next step in pipeline is Data Preparation, also referred to Data Preprocessing is transforming the raw data that was collected into a form that can be used in modeling. Since raw data can’t be used directly by machine learning models it has to be converted into form of numbers. For example, categorical data can’t be used directly in model. Though steps in Data Preparation can be similar for a number of problems but they can vary depending on data and problem. Nevertheless, these are some standard steps used:

These tasks include:
  1. Data Cleaning: Identifying and correcting mistakes or errors in the data.
  2. Feature Selection: Identifying those input variables that are most relevant to the task.
  3. Data Transforms: Changing the scale or distribution of variables.
  4. Feature Engineering: Deriving new variables from available data.
  5. Dimensionality Reduction: Creating compact projections of the data.

Transformed data is fed into models for training ans models are evaluated on the basis of a performance metric for evaluating the skill of a model, establishing a baseline or floor in performance to which all model evaluations can be compared.For example , while evaluating a model doing regression , Mean squared error could be used to evaluate the performance of model.After finalizing model , model is validated on a test data which it has not seen during the time of training. Finally , well performing model on validation is pushed into production systems and a monitoring and maintenance schedule is designed for the model.

Why ML pipeline is important?

Even though ML architectures contains a lot less code than other software applications, the the approach to keep all of the assets in one place makes sense. However , while scaling a ML architecture three significant problems arise :

  1. Volume: when deploying multiple versions of the same model, you have to run the whole workflow twice, even though the first steps of ingestion and preparation are exactly identical.
  2. Variety: when you expand your model portfolio, you’ll have to copy and paste code from the beginning stages of the workflow, which is inefficient and a bad sign in software development.
  3. Versioning: when you change the configuration of a data source or other commonly used part of your workflow, you’ll have to manually update all of the scripts, which is time consuming and creates room for error.

With the ML pipeline, each part of your workflow is abstracted into an independent service. Then, each time you design a new workflow, you can pick and choose which elements you need and use them where you need them, while any changes made to that service will be made on a higher level. A pipelining architecture solves the problems that arise at scale.
